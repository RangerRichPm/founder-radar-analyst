# 合规与伦理边界

## 3 条不可逾越的红线

### ❌ 红线 1：绝不卖保险
- **禁出现词**（12 词）：`保额` `报价` `投保` `险种` `产品链接` `销售` `加微` `私聊` `优惠` `限时` `保单` `退保`
- **禁止输出**：
  - 任何具体险种推荐（如"建议投保董监高责任险 500 万保额"）
  - 任何保险公司对接
  - 任何"购买链接" / "加微详询"
- **正确输出**：
  - "贵司在融资节点的关键人风险敞口约 60% → 建议建立关键人备份机制"
  - "如需就风险敞口做深度评估，可约时间交流"

### ❌ 红线 2：绝不主动找企业
- **节点发现名单是用户内部观察名单**，不向外推送
- **不接公开 marketing 渠道**（不做邮件群发、不做电话外呼、不做 LinkedIn 主动加好友）
- **触达永远是"建联"，不是"销售"**

### ❌ 红线 3：绝不深挖个人
- 创始人画像**只做 4 维度**：背景 / 当前重心 / 表达风格 / 挑战
- **不做的**：
  - 性格打分（如"内向/外向" "完美主义"）
  - 心理推测（如"压力大/焦虑"）
  - 家庭背景深挖（父母/配偶/子女）
  - 私生活（朋友圈/微博/小号）
  - 存储手机/邮箱/家庭住址

## 数据来源合规

### ✅ 可用
- 公开可访问信息（36氪 / IT桔子 / 企查查公开页 / 官网 / 公开新闻）
- 用户主动提供的信息
- WebSearch 公开搜索结果

### ❌ 不可用
- 登录态后内容（如需登录的 36氪付费文章）
- 付费墙后数据
- 直连 LinkedIn / 微信 / 脉脉（反爬重 + 合规风险）
- 抓取企业内部系统

## 风险提示模板（必须显示）

每张报告的底部强制显示：

```
⚠️ 免责声明
本报告基于 YYYY-MM-DD 公开信息推测，最终结论需进一步验证。
本系统输出的所有内容仅供风险观察与内部参考，不构成：
  - 任何保险建议或销售邀约
  - 任何投资建议
  - 任何法律或财务建议
```

## 创始人画像伦理

### ✅ 可做
- 公开演讲引述（注明来源 URL）
- 公开文章 / 博客 / 知乎回答
- 公司官网"关于我们"中的履历
- 公开访谈中的挑战自述

### ❌ 不可做
- 推断其心理状态
- 评价其"管理能力" / "领导力"等主观维度
- 储存个人手机/邮箱
- 监控私域账号

## 触达话术合规自检

`outreach.js` 内置 12 词违禁词表。生成话术后**强制自检**，命中任意一词需重新生成或人工干预：

```
forbiddenWords = [
    // 核心违禁词（compliance.md 权威定义）
    '保额', '报价', '投保', '险种', '产品链接',
    '销售', '加微', '私聊', '优惠', '限时',
    '保单', '退保',
    // 补充违禁词（明显保险关联）
    '保险', '保费', '理赔'
];
```

## 监管关注场景（特别注意）

如企业处于以下场景，**升级为高风险预警**（不直接拒绝，但必须明示）：

- 金融业务无牌照
- 数据跨境未做 GDPR/PDPA/PIPL 评估
- 涉及医疗/教育行业资质问题
- 近期有监管问询
- 涉及上市关键期

## 违规处理流程

如发现系统输出违反以上红线：

1. **立即停止输出**该内容
2. **提醒用户**违规风险
3. **建议替代方案**（如"建议删除"）
4. **记录到风险日志**（sessionStorage，便于审计）

## 行业敏感词（金融场景特别注意）

除 12 词违禁词外，金融场景还需避免：

- "年化收益" "保本" "刚兑"（投资属性）
- "内幕" "老鼠仓" "操纵"（市场操纵属性）
- "无风险" "稳赚" "躺赚"（夸大宣传）

如企业业务涉及金融牌照，**强制提示**用户核实相关资质。
