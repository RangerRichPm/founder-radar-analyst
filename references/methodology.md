# Founder Radar Analyst — 七阶段飞轮方法论

## 概述

Founder Radar 是面向**早期企业风险观察**的 7 阶段诊断飞轮。适用于：保险代销向风控咨询转型 / 早期投资尽调辅助 / 创业公司自检。

> 核心原则：**不卖保险**（先卖洞察）· **不找企业**（先找节点）· **不推产品**（先做诊断）

## 七阶段飞轮

```
节点发现 → 企业分析 → 创始人画像 → 风险推理 → 预警卡 → 触达话术 → 诊断对话
  ①          ②          ③          ④        ⑤        ⑥         ⑦
```

### ① 节点发现 (node-discovery)
- **目的**：从"关注赛道"反推处于关键经营节点的企业
- **5 类信号**：
  - 💰 融资：天使/A/B/C 轮完成
  - 🌍 出海：海外办公室、海外招聘、新市场进入、跨境数据
  - 📝 合同：大客户签约、政府项目、招投标
  - 📈 扩张：大规模招聘、新城市、新业务线
  - ⚠️ 事故/续保：诉讼、劳动争议、监管、保单到期
- **入口关键词**：`AI SaaS` `机器人` `跨境支付` `新能源` 等
- **产出**：候选企业名单（mock 或 WebSearch 接入）

### ② 企业分析 (company-intel)
- **9 维度**：行业、成立年、团队规模、总部、融资阶段、核心业务、近期事件、补充说明、保单状态
- **4 数据源**：表单输入、企查查（qcc-company connector）、天眼查（tyc-mcp）、WebFetch 官网

### ③ 创始人画像 (founder-intel)
- **4 维度标签**：
  1. 背景（学历 / 履历 / 创业史）
  2. 当前重心（融资 / 产品 / 团队 / 战略）
  3. 表达风格（公开演讲 / 文章 / 朋友圈）
  4. 挑战（公开访谈提及的卡点）
- **伦理红线**（绝不可越界）：
  - ❌ 不做性格打分、心理推测
  - ❌ 不深挖家庭背景
  - ❌ 不存储手机/邮箱等个人联系信息

### ④ 风险推理 (risk-engine + llm-reasoner)
- **节点 → 风险映射**（rule_hits）：
  - financing 节点：关键人 / 健康 / 董监高责任 / 股权 / 扩张
  - overseas 节点：跨境责任 / 海外雇佣 / 数据合规 / 海外产品 / 海外网安
  - contract 节点：合同责任 / 履约 / 公众责任
  - expansion 节点：雇主责任 / 劳动争议 / 核心员工流失 / 办公室搬迁
  - renewal 节点：保障缺口 / 保单失效
- **LLM 推理增强**（llm_added）：
  - 5 类隐性风险：决策疲劳、信任透支、合规盲区、现金流压力、组织失序
  - 7 行业严重度校准（金融 / AI / 医疗 / 教育 / 跨境等）
- **综合洞察**（llm_calibrated）：跨节点交叉风险（例：融资 + 扩张 → 估值倒挂）

### ⑤ 风险预警卡 (radar-card)
- **独立 HTML 页**（`radar.html`）：可独立打开、可分享 URL、可 PDF 导出
- **核心模块**：
  - 风险热区（按严重度高/中/低着色）
  - 行业案例（从 `data/industry-cases.json` 检索 3 条最相关）
  - AI 发现（LLM 推理的可解释文本）
  - 时效标识（"基于 YYYY-MM-DD 公开信息推测"）
  - 风险提示（强制底部 disclaimer）

### ⑥ 触达话术 (outreach)
- **4 渠道 7 话术**（默认生成 4 套，每套按渠道分发）：
  - A 共同朋友/校友/投资人介绍（首选，信任度最高）
  - B 微信（先做风险诊断再求介绍）
  - C LinkedIn（海外/外企背景创始人）
  - D 邮件（机构投资人 / 政府关系）
- **7 模板**：每渠道 1-2 套，覆盖冷启动 / 复访 / 拒绝挽回
- **违禁词自检**（12 词）：
  - 禁出现：`保额` `报价` `投保` `险种` `产品链接` `销售` `加微` `私聊` `优惠` `限时` `保单` `退保`
- **核心规则**：触达目的仅是"建联"，**绝不推销任何保险产品**

### ⑦ 风险诊断对话 (risk-chat)
- **七维框架**：创始人 / 团队 / 客户 / 产品 / 数据 / 出海 / 治理
- **对话模式**：用户选维度 chip → 自由提问 → LLM 推理结合 risk map 返回答案
- **终态产物**：可导出《企业风险地图》（七维评分 + 关键证据 + 推荐下一步）

## 工作流标准

调用 Skill 后，遵循以下顺序：

1. **确定赛道与节点**：用 ① 节点发现列出候选（如不熟悉，可直接进入 ② 手动录入）
2. **选择 1 家企业**：用 ② ③ 补全企业/创始人信息
3. **跑 ④ 风险推理**：自动生成规则命中 + LLM 增强
4. **输出 ⑤ 预警卡**：HTML + PDF 导出
5. **生成 ⑥ 触达话术**：先选首选渠道 → 再展开其他 3 套
6. **启动 ⑦ 风险对话**：根据用户回复逐步补全七维

## 边界声明

| 类别 | 可做 | 不可做 |
| --- | --- | --- |
| 数据 | 公开信息 + 用户输入 | 抓登录态/付费墙/直连 LinkedIn |
| 创始人 | 4 维度画像 + 来源 URL | 性格/心理/家庭深挖 + 存手机邮箱 |
| 输出 | 风险预警 + 触达 | 保险产品/保额/报价/链接 |
| 推荐 | 建联 + 风险科普 | 推销 + 销售转化话术 |
